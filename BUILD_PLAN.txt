================================================================================
  STEP 2 — BUILD THE FLASK BACKEND (app.py)
================================================================================

  Create a new file: C:\Users\rhoss\Desktop\Fraud-Finders\app.py

  This file needs these 4 endpoints:

  ── POST /api/run ──────────────────────────────────────────────────────────────
  Triggers the full 7-agent pipeline on the CSV.
  Returns 200 immediately, runs pipeline in background thread.

  ── GET /api/status ────────────────────────────────────────────────────────────
  Server-Sent Events (SSE) stream.
  Frontend connects to this and receives live log lines as agents run.
  Each event = one log line (agent name + message).
  Frontend displays these in the Agent Log terminal.

  ── GET /api/results ───────────────────────────────────────────────────────────
  Returns the final pipeline results as JSON:
  {
    "ring_accounts": ["AC-0001", "AC-0005", ...],
    "sink_accounts": ["AC-0002", ...],
    "sleeper_accounts": ["AC-0012"],
    "ring_exposure": 161751.23,
    "fraud_probability": 94.1,
    "severity": 9.4,
    "action_tier": "CRITICAL",
    "typologies": ["ML-027", "ML-008", "ML-041"],
    "sar_narrative": "...",
    "signal_breakdown": { ... }
  }

  ── GET /api/sar ────────────────────────────────────────────────────────────────
  Generates a SAR PDF using reportlab and returns it as a download.
  Content-Type: application/pdf
  Content-Disposition: attachment; filename="RingAlert_SAR_Report.pdf"

  Install requirements:
      pip install flask flask-cors reportlab --break-system-packages

  Run the server:
      python app.py
      → Server starts at http://localhost:5000

  STEP 3 — UPDATE THE FRONTEND TO CALL THE BACKEND

  Open fraudwatch_ui.html and make these 4 changes:

  3a. ▶ Run Pipeline button → calls POST /api/run
      Instead of the JavaScript animation, button triggers:

  3b. Agent Log terminal → connects to SSE stream
      After clicking Run, open EventSource to /api/status:
        const es = new EventSource('http://localhost:5000/api/status')
        es.onmessage = (e) => appendLog(JSON.parse(e.data))
      This makes the terminal show REAL agent output, not simulated.

  3c. Results display → calls GET /api/results after pipeline completes
      When SSE stream closes (pipeline done), fetch results:
        fetch('http://localhost:5000/api/results')
          .then(r => r.json())
          .then(data => updateDashboard(data))
      This replaces the hardcoded ACCOUNTS array with real data.

  3d. Download SAR button → calls GET /api/sar
      Instead of the JS blob download, redirect to:
        window.location.href = 'http://localhost:5000/api/sar'
      Browser downloads the real PDF.
