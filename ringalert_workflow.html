Track 02 — Fraud Watch  |  Multi-Agent Pipeline
The Product Idea

RingAlert

A bank's fraud ring stayed under every alert threshold — small transactions, circular flows, coordinated accounts. RingAlert maps 5,000 transactions into a knowledge graph and surfaces the hidden 12-account fraud ring that rules-based systems missed. Five AI agents, all handing off through Cognee's memory layer, deliver a plain-English case brief with a Bayesian risk score — in under 3 minutes per case.

Full AI Agent Workflow — All handoffs via Cognee
Cognee Memory Layer
Agent 1
DataIngestor
Step 1 — Find It
Reads the raw 5,000-transaction CSV. Normalises column names, converts amounts to float, drops nulls. Builds a structured entity list (accounts + transaction edges). Stores everything to Cognee via cognee.remember().
Cognee WRITE
→ remember() ⟵⟶ ← recall()
Input: track02_transactions.csv

Output stored to Cognee:
• List of all account IDs
• All 5,000 transaction records (normalised)
• Total row count + ingestion timestamp
↓
Reads from Cognee:
• All account entities
• All transaction records

Writes back to Cognee:
• Directed graph (A→B edges)
• Suspicion score per edge pair
• Micro-transaction flags
→ remember() ⟵⟶ ← recall()
Agent 2
GraphMapper
Step 2 — Rank It
Reads all entities from Cognee. Builds a directed relationship graph: who sent to whom, total flow, frequency, timing clusters. Calculates a suspicion score per edge pair. Flags micro-transactions (repeated amounts below $500). Stores enriched graph to Cognee.
Cognee READ + WRITE
↓
Agent 3
PatternDetector
Step 3 — Act On It
Reads the enriched graph from Cognee. Runs DFS cycle detection to find circular flows (A→B→C→A). Cross-references micro-transaction accounts and coordinated timing clusters. Flags each fraud ring with full evidence trail. Stores flagged rings to Cognee.
Cognee READ + WRITE Geodo research
→ remember() ⟵⟶ ← recall()
Detects 3 fraud patterns:

🔴 Circular flows — A→B→C→A money loops

🟡 Micro-transactions — repeated amounts just below $500 alert threshold

🟠 Coordinated timing — unrelated accounts transacting in synchronized bursts
↓
Reads from Cognee:
• All flagged ring objects
• Evidence trails

PyMC Bayesian model outputs:
• Fraud probability (e.g. 94%)
• 90% confidence interval (±3%)
• Downloadable narrative PDF ← R08
→ remember() ⟵⟶ ← recall()
Agent 4
RiskScorer
Step 4 — Explain It
Reads flagged rings from Cognee. Runs PyMC Bayesian inference — outputs "94% fraud probability ±3%" instead of a black-box score. Generates a downloadable narrative PDF per ring (required by Rule R08). Domain Expert owns the narrative text. Stores scores to Cognee.
Cognee READ + WRITE PyMC
↓
Agent 5
CaseBriefWriter
Step 5 — Show It
Reads ALL prior results from Cognee. Uses Claude API to write a plain-English 3-sentence case brief per ring: what happened, why it's suspicious, recommended action. Brief surfaces in the UI right panel. Zero SQL. Zero training needed to act on it.
Cognee READ Claude API
← recall() ↓ UI output
Final output to UI:

📋 Ring ID + risk level badge
📊 Risk score — 94% ±3% (PyMC)
🏦 Account list — the fraud ring members
📝 3-sentence brief — plain English
🚩 Flag for Review / ✓ Dismiss buttons
Judging Criteria — 5 pts each · 25 max · need 15 to qualify
1
Agents that work
Agent 1 runs directly on the provided CSV. If data loads and Cognee receives entities, you pass.
→ Build ingestion first. Nothing else matters if this fails.
2
Real collaboration via Cognee
Every agent reads what the previous wrote. cognee.remember() on write, cognee.recall() on read. Cognee IS the handoff.
→ No agent can skip Cognee. Not even one.
3
Matches your brief
Judged against YOUR OWN Step 0 promises. Underpromise, then deliver exactly that.
→ Submit Step 0 first. Write conservatively.
4
End user can use it
A judge picks it up cold at 4 PM. No CLI. No instructions. Big buttons. Plain English everywhere.
→ Upload CSV → Run → See fraud ring. That's the whole flow.
5
Explainable
Every output shows its reason. Risk score includes "why" (3 circular flows, 47 micro-tx, coordinated timing).
→ No black box outputs. Every decision has a label.
Tech Stack — Setup Commands
Cognee
MANDATORY
Memory layer for all agent handoffs. Graph-native — perfect for fraud networks. Every agent reads/writes here.
pip install cognee
→ cognee.ai · 14-day trial · Discord: discord.gg/5SHNNhe7t
Trupeer
MANDATORY (no video = DQ)
5-minute demo video. Auto-zooms on clicks, removes filler words, adds narration. Record at the end.
→ app.trupeer.ai/auth
$200 "Best Use of Trupeer" prize available
Geodo
MANDATORY
Domain Expert uses this to research real-world entities in the dataset. Web platform — no code required.
→ geodo.ai · 100 credits included
Sign up first, then request access via Google form
PyMC
OPTIONAL · $2,000 PRIZE
Bayesian fraud probability with confidence intervals. Outputs "94% ±3%" — defensible, explainable, impressive.
pip install pymc
Check hub.decision.ai for existing fraud skills first
Anthropic / Claude API
RECOMMENDED
Powers Agent 5 (CaseBriefWriter). Generates plain-English case briefs from Cognee data.
pip install anthropic
Bring your own API key. Groq free tier also works.
Lovable.dev + LingCode
RECOMMENDED
Lovable builds the UI from a prompt. LingCode is the browser IDE with Claude Code + Codex built in.
→ lovable.dev · lingcode.dev/try
No install needed — both are browser-based
